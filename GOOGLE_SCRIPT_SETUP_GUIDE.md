# 🚀 구글 스크립트 실시간 참여자 연동 설정 가이드

## 🎯 목표
구글 폼 제출 시 자동으로 구글 시트에 참여자 데이터를 기록하고, 웹사이트에 실시간으로 반영하는 시스템 구축

## 📋 1단계: 구글 스크립트 설정

### 1.1 Apps Script 접속
1. **구글 시트 접속**: https://docs.google.com/spreadsheets/d/1acRHfDOYSpkQGiYDShp9I6Fi4-SCa3IttAcsxOW3_KI/edit
2. **상단 메뉴** → **[확장 프로그램]** → **[Apps Script]** 클릭
3. **새 프로젝트 생성** (또는 기존 프로젝트 열기)

### 1.2 스크립트 코드 복사
1. **기본 `Code.gs` 파일 내용 모두 삭제**
2. **제공된 구글 스크립트 코드 전체 복사 붙여넣기**
3. **[저장]** 버튼 클릭 (Ctrl+S)

### 1.3 권한 승인
1. **[실행]** 버튼 클릭하여 권한 요청
2. **"권한 검토"** → **"고급"** → **"안전하지 않은 페이지로 이동"**
3. **"허용"** 클릭하여 모든 권한 승인

## 🔧 2단계: 트리거 설정

### 2.1 자동 트리거 설정 (권장)
1. **Apps Script 에디터**에서 **함수 선택**: `setupTriggersManually`
2. **[실행]** 버튼 클릭
3. **콘솔 로그 확인**: "트리거 설정 완료" 메시지 확인

### 2.2 수동 트리거 설정 (자동 실패 시)
1. **Apps Script 에디터** → **왼쪽 메뉴** → **[트리거]** 클릭
2. **[+ 트리거 추가]** 클릭
3. **설정값**:
   - **실행할 함수**: `onFormSubmit`
   - **이벤트 소스**: `스프레드시트에서`
   - **이벤트 유형**: `양식 제출 시`
4. **[저장]** 클릭

## 📊 3단계: 테스트 실행

### 3.1 기존 데이터 처리 (이지우님 데이터)
1. **Apps Script 에디터**에서 **함수 선택**: `testWithJiwooData`
2. **[실행]** 버튼 클릭
3. **콘솔 로그 확인**:
   ```
   이지우님 데이터로 테스트 시작...
   🎉 이지우님 데이터 처리 완료!
   📊 34기 현재 참여자: 1명 (남성 1명)
   ```

### 3.2 전체 기존 데이터 처리 (선택사항)
1. **함수 선택**: `testWithExistingData`
2. **[실행]** 버튼 클릭
3. **모든 기존 응답 데이터 자동 처리**

### 3.3 구글 시트 확인
**처리 후 구글 시트에 다음 시트들이 자동 생성됨**:
- **`Sessions`**: 모임별 참여자 수 현황
- **`Participants`**: 개별 참여자 상세 정보

### 4.1 구글 폼 테스트

2. **다른 기수 선택** (예: 35기 평택)

### 4.2 자동 처리 확인
**구글 폼 제출 후 즉시**:
1. **Apps Script 실행 로그** 확인 (Apps Script → 실행 → 로그)
2. **Sessions 시트** → 해당 기수 참여자 수 증가 확인
3. **Participants 시트** → 새 참여자 기록 추가 확인

### 4.3 웹사이트 반영 확인

- **해당 기수에서 "현재 X명" 증가 확인**

## 📈 5단계: Sessions 시트 CSV 게시

### 5.1 Sessions 시트 웹에 게시
1. **구글 시트**에서 **Sessions 시트 탭** 클릭
2. **[파일]** → **[공유]** → **[웹에 게시]**
3. **게시할 시트**: **Sessions** 선택
4. **출력 형식**: **CSV** 선택
5. **[게시]** + **"자동 재게시"** 체크

### 5.2 CSV URL 확인
**생성된 URL 형태**:
```
https://docs.google.com/spreadsheets/d/1acRHfDOYSpkQGiYDShp9I6Fi4-SCa3IttAcsxOW3_KI/pub?gid=SESSIONS_GID&single=true&output=csv
```
